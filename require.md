1. Функциональные требования
1.1. Захват экрана

Система должна:

захватывать весь экран, отдельное окно или выбранную область;
поддерживать множественные области захвата;
поддерживать области исключения (ignore zones);
работать с разными backend’ами захвата:
DXGI,
OpenGL,
Vulkan,
FFmpeg,
screen_capture_lite;
поддерживать режимы:
callback,
таймер,
гибридный режим;
передавать кадры в OCR-модуль с меткой времени.
1.2. OCR (распознавание текста)

Система должна:

находить текстовые области на кадре;
определять координаты текста;
разделять текст на логические / контекстные блоки;
извлекать:
текст,
bounding box,
цвет/фон,
параметры шрифта (если возможно);
поддерживать несколько OCR-движков;
поддерживать локальную работу без облака;
поддерживать предобработку изображения;
отслеживать изменения текста и не обрабатывать повторно неизменённые области.
1.3. Перевод текста

Система должна:

переводить текст локально без отправки данных в сеть;
поддерживать локальные модели перевода / LLM;
поддерживать:
автоопределение языка,
ручной выбор исходного языка,
выбор целевого языка;
переводить каждый блок отдельно;
объединять связанный текст в единый перевод;
поддерживать параллельный перевод нескольких блоков;
кэшировать уже переведённые строки.
1.4. Overlay / отображение перевода

Система должна:

отображать перевод поверх исходного приложения / экрана;
не блокировать ввод мыши и клавиатуры;
отображать перевод строго в области исходного текста;
поддерживать режимы фона:
фиксированный,
автоцвет,
градиент,
восстановленный фон;
поддерживать:
настройку шрифта,
цвета,
прозрачности;
поддерживать автостиль по исходному тексту;
поддерживать альтернативный режим вывода в отдельное окно.
1.5. Лаунчер / GUI

Система должна:

предоставлять интерфейс настройки модулей;
позволять:
выбирать backend захвата,
OCR-движок,
модель перевода,
стиль overlay;
настраивать:
области захвата,
интервалы обновления,
горячие клавиши;
сохранять / загружать профили настроек.
1.6. Плагинная система

Система должна:

поддерживать подключение модулей как плагинов;
иметь единый API для модулей;
поддерживать динамическую загрузку / выгрузку модулей;
уведомлять пользователя об отсутствующих зависимостях;
автоматически устанавливать недостающие компоненты.
2. Нефункциональные требования
2.1. Производительность

Система должна:

обеспечивать минимальную задержку между появлением текста и переводом;
работать в реальном времени;
минимизировать нагрузку на CPU / GPU;
использовать GPU memory copy при захвате;
избегать лишнего копирования данных;
пропускать устаревшие кадры при перегрузке.
2.2. Надёжность

Система должна:

корректно обрабатывать сбои модулей;
восстанавливать работу после ошибок;
не завершаться аварийно при отсутствии плагина;
логировать ошибки и предупреждения.
2.3. Масштабируемость

Система должна:

позволять добавлять новые:
OCR движки,
переводчики,
overlay backend’ы;
поддерживать расширение без изменения ядра.
2.4. Модульность

Система должна:

иметь слабую связанность модулей;
использовать стандартные интерфейсы;
позволять независимую разработку компонентов.
2.5. Кроссплатформенность

Система должна:

работать на Windows 10/11;
поддерживать Linux:
X11,
Wayland / PipeWire;
корректно работать с fullscreen приложениями.
2.6. Удобство использования

Система должна:

иметь понятный GUI;
поддерживать профили настроек;
поддерживать горячие клавиши;
иметь режим быстрого включения / отключения.
2.7. Безопасность и приватность

Система должна:

работать полностью локально;


2.8. Поддерживаемость

Система должна:

иметь логирование;
поддерживать debug режим;
поддерживать профилирование производительности;
иметь документацию API и модулей.
