# KODA.md — Контекст проекта

## Обзор каталога

Данный каталог содержит материалы для **дипломной работы** по разработке системы экранного перевода в реальном времени (Real-Time Screen Translator). Это не программный проект с исполняемым кодом, а **документация и проектная документация**.

### Назначение директории

Каталог хранит полный комплект проектной документации для создания кроссплатформенного приложения, которое выполняет:
- Захват области экрана
- Распознавание текста (OCR)
- Локальный перевод с использованием LLM
- Отображение перевода в виде оверлея поверх исходного приложения

## Ключевые файлы

### 1. ТЗ (1).md

**Техническое задание** на систему экранного перевода. Содержит:

- **Общее описание**: кроссплатформенное приложение (Windows/Linux) для захвата экрана, OCR, перевода и оверлея
- **Функциональные блоки (4 модуля)**:
  - Capture — захват экрана (DXGI, Vulkan, OpenGL, FFmpeg)
  - OCR — распознавание текста (Tesseract, PaddleOCR)
  - Translator — перевод (локальные LLM: LLaMA, Ollama, Marian NMT)
  - Overlay — отображение перевода поверх приложения
- **Архитектура асинхронного конвейера**: механизм «throttle + pending frame» (один активный кадр + один ожидающий)
- **Лаунчер и конфигурация**: Python/PyQt интерфейс для настройки
- **Технические требования**: C++17/20 для ядра, Python для лаунчера
- **Требования к производительности**: минимальное влияние на целевое приложение

- Система плагинов (PluginManager, IPlugin)

## Реализация (app/real_time_translator)

### Структура проекта

```
app/real_time_translator/
├── CMakeLists.txt              # Конфигурация сборки (CMake)
├── main.cpp                    # Точка входа
├── config/
│   └── default_config.json     # Конфигурация по умолчанию
├── core/
│   ├── CoreEngine.h/cpp        # Главный класс (ядро конвейера)
│   ├── Interfaces.h/cpp        # Базовые интерфейсы и структуры
├── modules/                    # Модули (stub-реализации)
│   ├── capture/NullCapture.h   # Захват (заглушка)
│   ├── ocr/StubOCR.h           # OCR (заглушка)
│   ├── translator/StubTranslator.h  # Перевод (заглушка)
│   └── overlay/StubOverlay.h   # Оверлей (заглушка)
└── utils/
    ├── LatestQueue.h/cpp       # Очередь throttle + pending frame
    ├── ThreadPool.h/cpp        # Пул потоков
    ├── Logger.h/cpp            # Логирование
    └── ConfigManager.h/cpp     # Управление конфигурацией
```

### Сборка и запуск

```bash
cd app/real_time_translator
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Debug
make -j4

# Запуск
./translator_app              # Обычный режим
./translator_app --verbose    # Подробный вывод
./translator_app --help       # Справка
```

### Особенности реализации

- **CoreEngine** — реализует механизм throttle + pending frame
- **ThreadPool** — параллельная обработка блоков перевода
- **LatestQueue** — перезаписываемая очередь (хранит только последний кадр)
- **LRU-кэш** — встроен в StubTranslator
- **Логирование** — с ротацией файлов
- **Обработка ошибок** — восстановление модулей при сбоях

## Дополнительные записи

При работе с этим проектом следует обратить внимание на:

1. **Асинхронный конвейер** — ключевая особенность архитектуры. Механизм «один активный + один pending» гарантирует отсутствие бесконечных отмен и предсказуемую задержку.

2. **Модульность** — каждый блок (Capture, OCR, Translator, Overlay) реализуется как плагин через единый интерфейс.

3. **Производительность** — главное требование: минимальное влияние на производительность целевого приложения (игры, редакторы, браузеры).

4. **Локальная работа** — перевод выполняется локально, без отправки данных в сеть (использование локальных LLM).
