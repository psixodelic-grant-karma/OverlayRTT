# Локальные переводчики для экранного переводчика

## Варианты переводчиков

### 1. MarianNMT (рекомендуется для игр)

**Преимущества:**
- Чистый C++ - минимальные накладные расходы
- Очень быстрый - подходит для real-time перевода в играх
- Малое потребление памяти (~50-100MB)
- Нет зависимости от Python/PyTorch

**Недостатки:**
- Требуется установка marian-nmt
- Модели нужно конвертировать в формат Marian

**Установка:**
```bash
# Ubuntu/Debian
sudo apt install marian-nmt

# Или из источника
git clone https://github.com/marian-nmt/marian-dev.git
cd marian-dev && mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release -DONLY_CMAKE_QT=OFF
make -j4 && sudo make install
```

**Модели:** Helsinki-NLP/opus-mt-*

---

### 2. Bergamot Translator (лучшее качество)

**Преимущества:**
- Высокое качество перевода
- Оптимизирован для перевода интерфейсов
- Использует ONNX - кроссплатформенно
- Поддержка многих языков

**Недостатки:**
- Требует Rust для сборки
- Среднее потребление памяти
- Медленнее MarianNMT

**Установка:**
```bash
# Установка через pip
pip3 install bergamot-translator

# Или из источника
git clone https://github.com/EdenWall/b2t.git
cd b2t && cargo build --release
```

**Модели:** edenwall/b2t-*

---

### 3. CTranslate2 (баланс)

**Преимущества:**
- Поддержка ONNX
- Хорошая скорость
- Простая интеграция

**Недостатки:**
- Требует Python
- Зависит от PyTorch
- Потребление памяти ~200-300MB

**Установка:**
```bash
pip3 install ctranslate2 transformers
```

---

### 4. LibreTranslate (не рекомендуется)

**Преимущества:**
- REST API
- Поддержка многих языков

**Недостатки:**
- Архитектурные ограничения
- Медленнее MarianNMT и Bergamot
- Требует отдельный сервер

## Рекомендация

**Для игр:** MarianNMT - минимальные накладные расходы, максимальная скорость.

**Для десктопных приложений:** Bergamot - лучшее качество при разумной производительности.

**Для MVP:** CTranslate2 - проще в настройке, но требует Python.
